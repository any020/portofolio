01 — Approach

Design, build, then test.

Working inside a complex, multi-module system means design decisions don't happen in a vacuum. Every choice has to account for how the system works, what developers can implement, and how real users interact with it — across roles, permissions, and contexts.

My process spans the full loop: from designing features in Figma with detailed dev notes, through implementation, to hands-on QA testing to make sure it actually works as intended.

1
Design

UI/UX design in Figma with annotated dev notes, interaction flows, and constraint documentation to bridge the gap between design intent and technical reality.

2
Handoff

Structured dev notes attached directly to frames — specifying behaviour, edge cases, and system logic so developers have exactly what they need.

3
Test

QA testing of built features across breakpoints and user roles — verifying that the implementation matches the design and works for the actual users of the system.
